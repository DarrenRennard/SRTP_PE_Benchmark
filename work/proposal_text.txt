（不超过 200 字） 
This project aims to develop a Power Electronics Design Agent Evaluation 
Framework to assess the capability of AI agents in performing realistic 
engineering design tasks. The framework focuses on constructing structured 
benchmark tasks and an automated evaluation pipeline, rather than developing a 
new AI model. Tasks include circuit design from specifications, simulation setup, 
waveform analysis, and engineering constraint verification. 
The evaluation will focus on two representative system types: DC-DC converters 
and traction inverters. The study will begin with DC-DC converter design, 
particularly flyback converters, as a controlled and well-understood problem 
setting. Based on this foundation, the framework will then be extended to traction 
inverter systems relevant to electric vehicle applications. 
The evaluation system will measure both engineering validity and task 
performance using metrics such as simulation success, functional correctness, and 
constraint satisfaction. By integrating Python-based orchestration with simulation 
tools such as MATLAB and PLECS, the project provides a reproducible and 
scalable environment for evaluating AI agents. The outcome will contribute to the 
development of reliable AI-assisted engineering systems and provide a foundation 
for future research in intelligent design automation.

二、项目背景、目的及意义

（简要说明项目背景、意义和实施必要性，研究现状和发展动态，不超过 1100 字） 
Modern power electronics plays a critical role in applications such as electric vehicles, 
renewable energy systems, and data centre infrastructure. However, engineering workflows in 
this field remain highly manual, iterative, and difficult to standardize. Tasks such as circuit 
design, simulation setup, and performance verification require significant expertise and time. 
At the same time, recent advances in AI agents and large language models have shown 
potential in automating complex technical workflows. Despite this progress, there is currently 
no standardized method to evaluate whether these agents can reliably perform engineering 
tasks in a physically meaningful and practically useful way. Many existing evaluations focus 
on language fluency rather than engineering correctness, which is insufficient in domains 
where incorrect outputs can lead to invalid or unsafe designs. 
This project addresses this gap by proposing a structured evaluation framework for AI agents 
in power electronics design. The objective is not to build a new AI system, but to create a 
rigorous benchmark environment where different agents can be tested on realistic engineering 
tasks. The framework emphasizes simulation-based validation and objective performance 
metrics to ensure that evaluation results reflect true engineering capability. 
To ensure feasibility and clarity, the research scope is explicitly limited to two representative 
system types: DC-DC converters and traction inverters. The study will first focus on DC-DC 
converter design, particularly flyback converters, as a controlled and well-understood 
problem setting. Based on this foundation, the framework will then be extended to traction 
inverter systems relevant to electric vehicle applications. This staged approach ensures both 
technical depth and practical feasibility within the project timeline. 
The significance of this project lies in three aspects. First, it introduces a systematic and 
engineering-grounded approach to evaluating AI agents, where physical correctness is 
essential. Second, it enables reproducible and scalable comparison between different AI 
systems using standardized tasks and metrics. Third, it contributes to the development of 
trustworthy AI tools that can assist engineers in real-world design tasks, particularly in 
safety-critical applications such as electric vehicle power systems. 
 
三、项目研究方案

The project consists of two main components: the design of a benchmark task suite and the 
development of an evaluation pipeline. 
The task suite will focus on flyback converter design and will be divided into four categories: 
1. Specification-to-design tasks: generating circuit designs based on given requirements 
such as voltage, power, and ripple constraints.  
2. Simulator setup tasks: producing executable simulation files (e.g., PLECS or MATLAB 
models).  
3. Waveform analysis tasks: interpreting voltage and current waveforms to identify system 
behaviour and issues.  
4. Engineering verification tasks: checking whether designs satisfy constraints such as 
efficiency, stability, and component limits.  
The evaluation pipeline will be implemented using Python and integrated with simulation tools. 
It will automate the process of: 
• Executing agent-generated outputs  
• Running simulations  
• Parsing results  
• Computing evaluation metrics  
Evaluation metrics include: 
• Format validity (structured output correctness)  
• Execution success (simulation runs successfully)  
• Functional correctness (desired performance achieved)  
• Constraint satisfaction (engineering limits respected)  
• Reasoning consistency (alignment between explanation and results)

The project timeline is as follows: 
Months 1 –2: Literature review on DC -DC converter and inverter design, such as flyback 
converters and traction inverters; tool familiarization and baseline AI agent deployment. 
Months 3–4: Design of the benchmark task suite for DC -DC converter tasks, including task 
categories, input/output formats, and validation rules. 
Months 5–6: Development of the evaluation pipeline, including simulation execution, result 
parsing, metric definition, and initial experiments on DC-DC converter tasks. 
Months 7 –8: Baseline testing with different agent setups; refinement of tasks and scoring 
methods; extension to traction inverter tasks. 
Months 9 –10: Systematic experiments, benchmark evaluation, comparative performance 
analysis on DC-DC converter and traction inverter tasks, and framework refinement. 
Months 11–12: Framework integration, documentation, final evaluation, and paper writing.

The project builds on existing knowledge in power electronics and AI systems, as well 
as available simulation tools such as MATLAB and PLECS. The student has strong 
programming skills and experience working with AI-based systems, which supports the 
implementation of the evaluation pipeline. 
However, challenges include limited prior work on standardized evaluation frameworks 
in this domain and the complexity of integrating simulation tools with AI outputs. These 
challenges will be addressed through incremental development and careful task design. 
The main innovations of this project include: 
1. Introducing a framework-based evaluation approach for AI in power 
electronics  
2. Using simulation-grounded metrics rather than subjective evaluation  
3. Combining engineering validation with AI task performance assessment  
4. Creating a reusable and scalable benchmark system  
Potential risks include integration difficulties and variability in AI outputs, but these can 
be mitigated through controlled task design and iterative refinement.
