# Documentation

This folder defines the research and evaluation conventions for the Power Electronics Design Agent Evaluation Framework.

## Core Documents

- [SCHEMA.md](SCHEMA.md) - conventions for research notes, evidence status, and red-team review.
- [evaluation_guidelines.md](evaluation_guidelines.md) - how to design, run, score, and report evaluations.
- [catalog.md](catalog.md) - index of current research notes grouped by field and status.
- [citations.md](citations.md) - master bibliography in IEEE style.
- [reference_research.md](reference_research.md) - findings from the reference repositories and the SRTP proposal.

## Reading Order

1. Read [SCHEMA.md](SCHEMA.md) before adding any research note.
2. Read [evaluation_guidelines.md](evaluation_guidelines.md) before adding a benchmark.
3. Use [catalog.md](catalog.md) to find current notes and avoid duplicating claims.
4. Add all new sources to [citations.md](citations.md).

## Working Rule

Claims about engineering performance must be tied to executable evidence where possible. A convincing explanation is useful, but it is not enough to pass a benchmark.
